Java Functional Programming

1. What is Functional Programming in Java?

El Functional Programming es un paradigma que trata la computación como la evaluación de funciones, evitando el estado mutable y los efectos secundarios.

Desde Java 8, Java soporta este paradigma mediante:

  • Lambdas
  • Functional Interfaces
  • Streams API
  • Optional

2. Functional Interfaces

Una Functional Interface es una interfaz con exactamente un método abstracto. 

@FunctionalInterface
public interface Calculator {
  int calculate(int a, int b);
}

Puede ser implementada usando una lambda: 

Calculator sum = (a, b) -> a + b;

3. Why Create Custom Functional Interfaces?

1️⃣ Domain-specific behavior

Las interfaces estándar (Function, Predicate, etc.) no siempre expresan claramente la intención del negocio. 

@FunctionalInterface
public interface PriceCalculator {
  BigDecimal applyDiscount(BigDecimal price);
}

2️⃣ Readability & Expressiveness

El código se vuelve más legible y autoexplicativo. 

processOrder(price, discountCalculator);

3️⃣ Checked Exceptions

Las interfaces funcionales estándar no permiten checked exceptions. 

@FunctionalInterface
public interface FileProcessor {
  String process(Path path) throws IOException;
}

4️⃣ Reusability & Testing

Facilitan la inyección de comportamiento y el testing unitario.

4. Practical Functional Programming Examples

Example 1: Filtering and Mapping with Streams

List<String> names = List.of("Ana", "Carlos", "Bob");

List<String> result = names.stream()
  .filter(name -> name.length() > 3)
  .map(String::toUpperCase)
  .toList();

Example 2: Behavior Parameterization

public void process(List<Integer> list, Predicate<Integer> rule) {
  list.stream()
      .filter(rule)
      .forEach(System.out::println);
}
process(numbers, n -> n % 2 == 0);

Example 3: Strategy Pattern (Functional Style)

Map<String, Function<Integer, Integer>> operations = Map.of(
  "double", x -> x * 2,
  "square", x -> x * x
);

operations.get("double").apply(5);

Example 4: Optional to Avoid Null Checks

Optional<User> user = findUser();

user.map(User::getEmail)
    .ifPresent(System.out::println);

Example 5: Immutable Transformations

List<Integer> doubled =
  numbers.stream()
         .map(n -> n * 2)
         .toList();

5. Common Interview Questions

Q1: Why use functional programming in Java?
Mejora la legibilidad, reduce el código boilerplate, facilita la concurrencia y promueve código más declarativo.
Q2: Why create custom functional interfaces?
Para expresar mejor reglas de negocio, soportar checked exceptions y mejorar la claridad del código.
Q3: Difference between Function and Predicate?
Function transforma un valor y retorna otro. Predicate evalúa una condición y retorna boolean.
Q4: Are lambdas thread-safe?
Sí, siempre que no accedan a estado mutable compartido.
Q5: Can a functional interface have default methods?
Sí, siempre que tenga un solo método abstracto.

6. Summary

Java Functional Programming permite escribir código más limpio, modular y expresivo. Crear interfaces funcionales personalizadas es una herramienta poderosa para modelar reglas de negocio y aplicar patrones de diseño de forma moderna.
