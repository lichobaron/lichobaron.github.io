Servicios Más Comunes de AWS

¿Qué es AWS?

[Amazon Web Services](chatgpt://generic-entity?number=0) (AWS) es una plataforma de computación en la nube que ofrece servicios bajo demanda como cómputo, almacenamiento, bases de datos, redes y seguridad.

  • Modelo pay-as-you-go
  • Alta disponibilidad y escalabilidad
  • Amplio ecosistema de servicios

Compute (Cómputo)

EC2 (Elastic Compute Cloud)

Máquinas virtuales en la nube. Permite control total del sistema operativo.

  • Escenarios: aplicaciones legacy, servidores personalizados
  • Se cobra por tiempo de uso
  • Requiere gestión del sistema operativo

Lambda

Servicio serverless para ejecutar código sin administrar servidores.

  • Escenarios: APIs, jobs, eventos
  • Escala automáticamente
  • Pago por ejecución

Storage (Almacenamiento)

S3 (Simple Storage Service)

Almacenamiento de objetos altamente duradero.

  • Escenarios: archivos, backups, hosting estático
  • Durabilidad del 99.999999999%
  • No es un sistema de archivos tradicional

EBS (Elastic Block Store)

Volúmenes de almacenamiento para instancias EC2.

  • Se comporta como un disco duro
  • Debe estar en la misma AZ que la EC2

Databases

RDS (Relational Database Service)

Servicio administrado para bases de datos relacionales (MySQL, PostgreSQL, Oracle, SQL Server).

  • Backups automáticos
  • Alta disponibilidad con Multi-AZ
  • No es serverless

DynamoDB

Base de datos NoSQL serverless, key-value y document.

  • Latencia de milisegundos
  • Escala automáticamente
  • Ideal para microservicios

Networking

VPC (Virtual Private Cloud)

Red virtual aislada dentro de AWS.

  • Control de subnets, IPs y routing
  • Base de toda arquitectura en AWS

Elastic Load Balancer (ELB)

Distribuye tráfico entre múltiples destinos.

  • Alta disponibilidad
  • Soporta HTTP, HTTPS y TCP

Security & Identity

IAM (Identity and Access Management)

Gestión de usuarios, roles y permisos.

  • Principio de mínimo privilegio
  • Roles recomendados sobre access keys

KMS (Key Management Service)

Servicio para manejo de claves de cifrado.

  • Cifrado de datos en reposo
  • Integrado con otros servicios AWS

Monitoring & Messaging

CloudWatch

Métricas, logs y alarmas para servicios AWS.

  • Monitoreo centralizado
  • Alarmas automáticas

SQS (Simple Queue Service)

Servicio de colas para desacoplar sistemas.

  • Alta durabilidad
  • Ideal para procesamiento asíncrono

CI/CD & Infrastructure

CloudFormation

Infraestructura como código (IaC).

  • Templates declarativos
  • Versionamiento de infraestructura

CodePipeline

Automatización de pipelines CI/CD.

  • Integración con GitHub
  • Deploy continuo

Detalles Clave para Entrevistas

  • Lambda ≠ EC2 (serverless vs servidores)
  • S3 ≠ EBS (objeto vs bloque)
  • DynamoDB ≠ RDS (NoSQL vs SQL)
  • IAM es global, no regional
  • Todo corre dentro de una VPC

Summary

AWS ofrece un amplio conjunto de servicios para construir aplicaciones escalables y seguras. Comprender los servicios más comunes y sus casos de uso es fundamental para diseñar arquitecturas en la nube y para entrevistas técnicas orientadas a Cloud.
